## 🛠️ Collaboration Guidelines

1. **Branching Strategy**: Each group should create their own branches for development (e.g., `group1/feature-name`).
2. **Commit Messages**: Keep commit messages clear and descriptive.
3. **Pull Requests**: When a feature is ready, submit a Pull Request (PR) for review.
4. **Documentation**: Ensure your group's `README.md` is kept up to date with your latest progress.
